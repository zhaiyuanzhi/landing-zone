## 1.2 说明

- 本章仅保留**直接进入四大类资产收益建模或风险建模**的外部原始指标。
- 无风险利率单独补入 `CGB_1Y`，用于组合优化、超额收益计算、夏普比率、Black-Litterman / MVO 等模型中的 `r_f`。
- 现金类在中国市场缺少统一、长期、稳定可得的现金总回报指数，因此当前采用**中国货币市场基金月平均七日年化收益率**作为最稳妥近似。
- 固收类保留一条**总回报口径主腿**与一组**利率 / 信用风险分解指标**，便于同时支持收益回测与信用利差分析。
- 股票类同时保留**总回报口径**与**价格口径**，分别对应长期配置回报建模与价格因子建模。
- 另类资产中，黄金与商品共同承担分散作用；如后续可得商品期货总回报指数，可进一步优化方案 A 的另类商品口径。

## 1.3 加工方式（非采购项）

以下加工结果均由上述已采购原始指标在系统内计算，不属于外部采购项：

- `r_f = CGB_1Y`
- `r_cash_A ≈ MMF_7D_YIELD_M / 12`
- `CREDIT_SPREAD_3Y_AA = AA_CREDIT_YIELD_3Y - CGB_3Y`
- `r_equity_A = ln(CSI300_TR_t / CSI300_TR_{t-1})`
- `r_equity_B = ln(CSI300_t / CSI300_{t-1})`
- 另类资产收益率序列由原始价格序列在系统内统一生成；如进入月度回测，可在系统内聚合为月频收益序列。

## 1.4 方案 A / 方案 B（简要说明）

> 说明：本节只做简要解释，不改变本章“采购清单以原始指标为主”的定位。四大类资产定义为：现金类强调流动性管理，固收类强调稳健收益，股票类承担长期进攻，另类资产承担分散风险与组合优化。

### 1.4.1 方案 A：Total Return（投资回报口径）

- 无风险利率：`CGB_1Y`
- 现金类：`MMF_7D_YIELD_M`
- 固收类：`CBOND_NEW_COMPOSITE_WEALTH`
- 股票类：`CSI300_TR`
- 另类-黄金：`AU9999`
- 另类-商品：`NHCI`

### 1.4.2 方案 B：Price Factor（价格因子口径）

- 无风险利率：`CGB_1Y`
- 固收利率 / 信用分解：`CGB_3Y`、`AA_CREDIT_YIELD_3Y`
- 股票类：`CSI300`
- 另类-黄金：`AU9999`
- 另类-商品：`NHCI`

## 1.5 相关性矩阵、预期收益与无风险利率的多窗口运算规则

### 1.5.1 运算对象与边界

- 本节面向四大类资产配置建模的参数层，统一回答三类核心问题：**相关性矩阵怎么计算、长期预期收益 `μ` 怎么估计、无风险利率 `r_f` 怎么进入模型**。
- 风险资产统一按 **5 条资产腿** 处理：
  - 现金：`MMF_7D_YIELD_M`
  - 固收：`CBOND_NEW_COMPOSITE_WEALTH`
  - 股票：`CSI300_TR`（方案 A 主口径）
  - 黄金：`AU9999`
  - 商品：`NHCI`
- 无风险利率单独采用 `CGB_1Y`，作为 `r_f` 序列处理，**不进入 risky assets 相关性矩阵 / 协方差矩阵**。
- 方案 B 中的 `CGB_3Y`、`AA_CREDIT_YIELD_3Y`、`CSI300` 更适合用于利率冲击、信用利差、价格因子解释，不与方案 A 的资产总回报主腿混合运算同一套配置矩阵。
- 因此系统内建议保留 **两套矩阵体系**：
  - **方案 A：资产配置主矩阵**，用于组合优化、风险预算、回测、Black-Litterman / MVO。
  - **方案 B：解释型因子矩阵**，用于风险归因、市场解释、战术分析。

### 1.5.2 统一频率与收益率口径

- 正式配置矩阵建议统一为**月频**运算。
- 所有进入矩阵的序列必须满足：**同频、同口径、同窗口**。
- 方案 A 推荐采用以下月收益率定义：

```text
r_cash_t      ≈ (1 + MMF_7D_YIELD_M_t)^(1/12) - 1
r_bond_t      = ln(CBOND_NEW_COMPOSITE_WEALTH_t / CBOND_NEW_COMPOSITE_WEALTH_{t-1})
r_equity_t    = ln(CSI300_TR_t / CSI300_TR_{t-1})
r_gold_t      = ln(AU9999_t / AU9999_{t-1})
r_cmdty_t     = ln(NHCI_t / NHCI_{t-1})
```

- 上述 5 条资产腿共同构成风险资产收益矩阵：

```text
R_t = [r_cash_t, r_bond_t, r_equity_t, r_gold_t, r_cmdty_t]
```

- 方案 B 若用于解释型矩阵，则应统一转换为“收益率 / 变化量”口径，而非直接拿 level 算相关：

```text
r_equity_B_t        = ln(CSI300_t / CSI300_{t-1})
Δy_cgb3y_t          = CGB_3Y_t - CGB_3Y_{t-1}
Δspread_aa3y_t      = (AA_CREDIT_YIELD_3Y - CGB_3Y)_t - (AA_CREDIT_YIELD_3Y - CGB_3Y)_{t-1}
r_gold_t            = ln(AU9999_t / AU9999_{t-1})
r_cmdty_t           = ln(NHCI_t / NHCI_{t-1})
```

### 1.5.3 相关性矩阵与协方差矩阵的计算规则

- 方案 A 的正式配置矩阵，基于 `R_t` 直接计算：

```text
Corr(W) = corr(R; window = W)
Cov(W)  = cov(R; window = W)
```

- 其中：
  - `Corr(W)` 为窗口 `W` 下的相关性矩阵；
  - `Cov(W)` 为窗口 `W` 下的协方差矩阵；
  - 若使用月频收益率，则年化协方差矩阵为：

```text
Cov_annual(W) = 12 × Cov_monthly(W)
```

- 相关性矩阵与协方差矩阵只对 **5 条风险资产腿** 计算，**不包含 `r_f`**。
- 正式配置场景优先使用方案 A，不建议把总回报指数、价格指数、收益率水平、信用利差等不同口径序列混入同一套配置矩阵。

### 1.5.4 多窗口设置规则

考虑中国市场与全球市场均存在明显的制度切换与危机阶段，仅保留 36M / 60M / 120M 不足以覆盖 2015 年中国股灾与 2008 年全球金融危机。因此，参数窗口建议分为三层：

#### （1）短中期观察窗口

- `12M`
- `24M`
- `36M`

用途：
- 观察近端 regime 变化；
- 捕捉相关性漂移；
- 支持战术监控与敏感性分析。

#### （2）主配置窗口

- `60M`
- `120M`

用途：
- 作为正式配置、回测、风险预算的主输入；
- 在稳定性与现实性之间取得平衡。

建议：
- `60M` 作为**默认标准窗口**；
- `120M` 作为**中长期锚**。

#### （3）危机校准窗口

- `180M`
- `240M`

用途：
- 覆盖更完整的压力样本；
- 将 2015 年中国股灾、2008 年全球金融危机等危机阶段纳入参数校准；
- 检查当前窗口是否低估极端市场下的共振风险。

### 1.5.5 长期预期收益 `μ` 的多窗口估计规则

- 预期收益 `μ` 可以、也应当按不同窗口分别估计。
- 对于任一窗口 `W`，月频历史均值定义为：

```text
μ(W) = mean(R_t; window = W)
```

- 若需年化，可采用：

```text
μ_annual(W) = 12 × μ_monthly(W)
```

- 系统建议至少保留以下参数：

```text
μ_12M, μ_24M, μ_36M, μ_60M, μ_120M, μ_180M, μ_240M
```

- 各窗口角色建议如下：
  - `μ_12M`、`μ_24M`：反映短期趋势，但噪声较大，只适合做修正参考；
  - `μ_36M`、`μ_60M`：实务基准；
  - `μ_120M`、`μ_180M`、`μ_240M`：长期锚，用于平滑短期噪声与补足危机样本。
- 投顾与配置场景下，不建议机械地只使用单一历史均值作为未来收益预期。更稳妥的做法是保留多窗口 `μ`，并在优化器或上层配置逻辑中进行融合。
- 默认建议：

```text
μ* = 融合(μ_60M, μ_120M)
```

必要时可再结合 `μ_180M` 或 `μ_240M` 做长期校准。

### 1.5.6 无风险利率 `r_f` 的运算规则

- 无风险利率采用 `CGB_1Y`，先统一转换为月频无风险收益率序列：

```text
r_f,t ≈ (1 + CGB_1Y_t)^(1/12) - 1
```

- 若原始数据为百分比口径，需先除以 100 后再计算。
- `r_f` 的角色是：
  - 计算超额收益；
  - 计算夏普比率；
  - 进入 MVO / Black-Litterman / 切点组合等优化器。
- `r_f` **不是相关性矩阵的一部分**，也不单独构造“无风险矩阵”。

#### （1）历史回测中的用法

对任一风险资产 `i`：

```text
r_excess,i,t = r_i,t - r_f,t
```

即按时间点一一对应地从资产收益中扣除当期无风险收益率。

#### （2）长期参数中的用法

可按不同窗口保留平均无风险利率参数：

```text
rf_avg_12M, rf_avg_24M, rf_avg_36M, rf_avg_60M, rf_avg_120M, rf_avg_180M, rf_avg_240M
```

其含义不是“无风险矩阵”，而是：
- 某一窗口下的平均无风险收益率；
- 用于 Sharpe、资本市场线、长期配置比较与优化器输入。

#### （3）默认建议

- 历史回测：使用逐期 `r_f,t`；
- 当前优化器：优先使用 `rf_avg_12M` 或最近时点 `r_f`；
- 长期研究：结合 `rf_avg_60M`、`rf_avg_120M`、`rf_avg_240M` 观察不同利率 regime 下的长期锚。

### 1.5.7 推荐的系统输出对象

围绕每个窗口 `W ∈ {12M, 24M, 36M, 60M, 120M, 180M, 240M}`，系统建议输出四类核心参数：

```text
Corr(W)
Cov(W)
μ(W)
rf_avg(W)
```

其中：
- `Corr(W)`：5 条风险资产腿的相关性矩阵；
- `Cov(W)`：5 条风险资产腿的协方差矩阵；
- `μ(W)`：5 条风险资产腿的预期收益向量；
- `rf_avg(W)`：对应窗口下的平均无风险收益率。

### 1.5.8 默认落地建议

- **正式配置主输入**：
  - `Cov_60M`
  - `μ* = 融合(μ_60M, μ_120M)`
  - `rf_avg_12M` 或最近时点 `r_f`
- **敏感性 / 漂移监控**：
  - `Corr_12M`
  - `Corr_24M`
  - `Corr_36M`
  - 对比 `Corr_60M`
- **危机校准检查**：
  - `Cov_180M`
  - `Cov_240M`

若当前 `60M` 参数明显低于 `180M / 240M` 危机校准窗口所反映的共振强度，则应提示当前配置矩阵可能低估极端市场下的组合风险。

## 1.6 一句话总结
