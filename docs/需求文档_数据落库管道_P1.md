## 1. 背景与目标

智能投顾3.0 依赖约 100 个外部采购指标驱动月报生成与量化建模。本期需求聚焦**最小可落地闭环**：将采购到的原始文件写入 MySQL 数据库各 Layer 1 表，为后续数据加工、聚合与 LLM 消费奠定基础。

### 4.2 新增表（本期补充 DDL）

数据库设计方案中 D-01a / D-01b 两项待决策，本期明确为：**新建 `benchmark_monthly` 和 `benchmark_daily` 两张独立表**，与宏观月频数据语义隔离。

#### 4.2.1 `benchmark_monthly` — 量化基准月度序列（Layer 1g）

覆盖范围：
- 4.3 原始月度基准：CBOND_COMPOSITE_FULLPRICE、511520、FDR007、807400
- 4.4 月频基准指数：HFRIMI、HFRIEMNI、HFRIDSI、朝阳永续5个私募指数

```sql
CREATE TABLE benchmark_monthly (
    id               BIGINT        NOT NULL AUTO_INCREMENT,
    ticker           VARCHAR(40)   NOT NULL  COMMENT '如 CBOND_COMPOSITE_FULLPRICE / HFRIMI',
    stat_month       CHAR(7)       NOT NULL  COMMENT '统计月份 YYYY-MM',
    px               DECIMAL(18,6) NULL      COMMENT '净值/价格/指数点位',
    ret_m            DECIMAL(12,6) NULL      COMMENT '月度收益率（小数，如 0.012）',
    data_source      VARCHAR(60)   NULL      COMMENT '如 Wind / HFRI / 朝阳永续',
    created_at       TIMESTAMP     DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE KEY uk_ticker_month (ticker, stat_month),
    INDEX idx_stat_month (stat_month),
    INDEX idx_ticker     (ticker)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
  COMMENT='量化基准月度序列 — Layer 1g（4.3原始月度基准 + 4.4月频指数）';
```

> px 与 ret_m 至少填一个；若供应商仅提供收益率则 px 为 NULL，反之亦然。

#### 4.2.2 `benchmark_daily` — 多资产配置基准指数日频（Layer 1h）

覆盖范围：
- 4.4 日频基准：Wind 基金指数5个、Bloomberg 债券指数5个、MSCI 权益指数3个、CME 期货2个

```sql
CREATE TABLE benchmark_daily (
    id               BIGINT        NOT NULL AUTO_INCREMENT,
    ticker           VARCHAR(40)   NOT NULL  COMMENT '如 885001.WI / LEGATRUU / MXWD / GC.CMX',
    trade_date       DATE          NOT NULL  COMMENT '交易日',
    close_price      DECIMAL(18,6) NULL      COMMENT '收盘价/净值/指数点位',
    ret_d            DECIMAL(12,6) NULL      COMMENT '日度收益率（可选，小数）',
    data_source      VARCHAR(60)   NULL      COMMENT '如 Wind / Bloomberg / CME',
    created_at       TIMESTAMP     DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE KEY uk_ticker_date (ticker, trade_date),
    INDEX idx_trade_date (trade_date),
    INDEX idx_ticker     (ticker)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
  COMMENT='多资产配置基准指数日频 — Layer 1h（4.4日频指数）';
```

## 5. 数据落库需求（ETL Loaders）

每个 Loader 对应一个 Python 模块，路径约定为 `src/etl/load_{name}.py`，支持 `--file` 或 `--month` 参数。

### L1 — NBS 统计局月频宏观数据

**目标表**：`macro_monthly`

| 项目 | 说明 |
|------|------|
| 数据源 | NBS（国家统计局）`data/raw/nbs/` |
| 文件格式 | `.xlsx`，前若干行为标题元数据，需跳过 |
| 历史窗口 | 近20年（约 2006-2026） |
| 触发频率 | 每月15日后手动或定时触发 |

**指标映射**（indicator_code → 原始列名）：

| indicator_code | 指标名称 | frequency |
|---------------|---------|-----------|
| `CPI_YOY` | 居民消费价格指数同比 | monthly |
| `PPI_YOY` | 工业生产者出厂价格指数同比 | monthly |
| `PMI_MANU` | 制造业PMI | monthly |
| `PMI_SERV` | 非制造业PMI | monthly |
| `INDUSTRY_YOY` | 工业增加值同比 | monthly |
| `AUTO_RETAIL_SALES_YOY` | 汽车零售销售同比 | monthly |
| `RETAIL_SALES_YOY` | 社会消费品零售总额同比 | monthly |
| `INVESTMENT_FA_YOY` | 固定资产投资同比 | monthly |
| `INVESTMENT_FA_PRIVATE_YOY` | 民间固定资产投资同比 | monthly |
| `POWER_GEN_YOY` | 发电量同比 | monthly |
| `RESID_HOUSE_AREA_CUMULATIVE_YOY` | 住宅销售面积累计同比 | monthly |
| `RESID_HOUSE_SALES_CUMULATIVE_YOY` | 住宅销售额累计同比 | monthly |
| `EXPORT_CUMULATIVE_YOY` | 出口累计同比 | monthly |
| `IMPORT_CUMULATIVE_YOY` | 进口累计同比 | monthly |
| `TRADE_TOTAL_CUMULATIVE_YOY` | 进出口总额累计同比 | monthly |
| `URBAN_SURVEY_UNEMPLOYMENT` | 城镇调查失业率 | monthly |
| `URBAN_YOUTH_UNEMPLOYMENT` | 城镇青年失业率 | monthly |
| `REAL_GDP_QOQ` | 实际GDP环比（季调） | quarterly |

> **REAL_GDP_QOQ 说明**：季度数据写入 `macro_monthly`，`stat_month` 取季度末月（Q1→03, Q2→06, Q3→09, Q4→12）。

**写入规则**：
- `stat_month` 格式 `YYYY-MM`
- `source_tag='NBS_OFFICIAL'`
- `data_version` 默认1；统计局修订时追加 version+1，不覆盖原行
- 字符串空值、`'NaN'`、`'-'` 均写 `NULL`，禁止填0或均值
- 月份重复（已存在相同 code+month+version）跳过，记录日志
